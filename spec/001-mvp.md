# Microblog Spec

## Overview

A statically generated microblog written via `jrnl`, with a paginated feed and full-text search powered by Pagefind. No server required.

---

## Writing Interface

- **Tool:** `jrnl`
- **Format:** Plain text with inline Markdown
- **Entry anatomy:** First sentence → title; remainder → body
- **Tags:** jrnl native tags (`@tag`)
- **Export:** `jrnl --export json > entries.json`

---

## Build Pipeline

```
jrnl --export json > entries.json
python build.py
pagefind --site dist/
```

`build.py` reads `entries.json` and writes the `dist/` directory.

---

## Site Structure

```
dist/
  index.html          # page 1 of feed
  page/
    2/index.html
    3/index.html
    ...
  entry/
    20260327T143000/index.html # individual entry page
  search/
    index.html        # search UI
  assets/
    style.css
```

---

## Pages

### Feed (`/`, `/page/N/`)

- Entries sorted newest-first
- 20 entries per page
- Each entry shows: date, title (linked), full rendered body, tags
- Previous / Next pagination links

### Entry (`/entry/<id>/`)

- ID is the entry datetime normalized: `20260327T143000`
- Full rendered Markdown body
- Date and tags displayed
- Link back to feed

### Search (`/search/`)

- Single `<input>` field
- Results rendered client-side by Pagefind JS
- Each result shows: title, date, excerpt with highlighted match

---

## Search

- **Engine:** [Pagefind](https://pagefind.app/)
- Indexes `dist/` after build
- Runs entirely in-browser — no server
- Index stored in `dist/_pagefind/`

Pagefind reads `data-pagefind-body` attribute to scope indexing to entry content.

---

## Markdown Rendering

- **Library:** `mistune` (Python, zero deps)
- Applied to each entry body during build
- Output is escaped HTML embedded in templates

---

## Templates

Minimal inline HTML templates in `build.py` (no template engine dependency). Single CSS file, no JS except Pagefind.

---

## Publish

```
rsync -av dist/ user@host:/var/www/blog/
# or: push dist/ to GitHub Pages / Netlify
```

---

## Out of Scope (for now)
